Jadi, 100% aman. Tapi, HTML Template Engine ini nggak bisa berjalan dengan sendirinya. Dia harus bernaung sama suatu wadah supaya bisa jalan. Wadah itu bernama Static Site Generator. Ada banyak Static Site Generator, di antaranya adalah Jekyll, Hugo, Eleventy, hingga Mini SSG Zen. ![Maskot Mini SSG Zen](https://raw.githubusercontent.com/mzaini30/mini-SSG/HEAD/tsukuyomi-komoe.png) Aku sebelumnya pakai Mini SSG Zen untuk develop file-file HTML, termasuk develop website ini. Namun, suatu masalah muncul. Kalau yang namanya Static Site Generator itu, dia generate semua file template dulu menjadi HTML, baru kemudian reload. Misalnya, aku punya 5 file Nunjucks seperti ini: ``` base.html index.html about.html galeri.html portfolio.html ``` Maka, SSG (static site generator) akan mengolah sintaks-sintaks Nunjuck di dalam `base.html`, lalu di `index.html`, kemudian di `about.html`, lalu di `galeri.html`, dan `portfolio.html`, baru kemudian `localhost` akan reload. Kalau cuma 5 sih nggak masalah. Waktu generatenya nggak terlalu lama. Nah, gimana kalau 100 file? Itulah masalahnya. Kalau misalnya SSG memerlukan waktu 1 detik untuk generate 1 file Nunjucks. Maka, SSG memerlukan waktu 1 menit dan 40 detik untuk generate 100 file, baru kemudian `localhost` reload. Itu kalau misalnya file-file Nunjucks di dalamnya nggak saling import. Kalau misalnya saling import, maka 1 file aja bisa makan waktu 3 detik. Kalau 100 file, maka kita perlu waktu 300 detik (5 menit) untuk generate semuanya, baru `localhost` reload. Bayangkan deh, kamu cuma nambahin titik di `index.html`, lalu kamu perlu nunggu 5 menit untuk melihat hasilnya di `localhost`. Bayangkan... Itu Indomie sepiring udah habis dah, plus bisa nambah lagi. Di sinilah yang mengganjel benakku. Kalau misalnya aku menggunakan SSG, maka perlu waktu lama untuk melihat hasil modify tiap file template yang aku buat. Maka, aku pun memiliki konsep seperti ini untuk tools yang akan aku buat: > Generate semua file hanya saat `build`. Maka, saat `dev`, aku nggak boleh nunggu semua generate. Nah, lalu gimana caranya? Aku bingung. Akhirnya, aku pun nemu solusinya, yaitu saat mode `dev`, maka dia akan menjalankan server Express. Mantap... Untungnya kan, Nunjucks itu kompatible dengan server Express. Kurancanglah toolsnya hingga akhirnya terbentuklah Unmei... ![Fotonya Unmei](https://raw.githubusercontent.com/mzaini30/unmei/HEAD/unmei.jpeg) Seperti yang aku bilang di atas bahwa dengan Unmei, saat `dev`, dia menggunakan server Express. Jadi, reloadnya ya nunggu generate satu file itu aja, misalnya `index.html`, nggak perlu nunggu generate 100 file template yang kita miliki. Nah, lalu pas `build`, dia akan generate semuanya, tapi nggak masalah buat kita karena proses build itu kan dilakukan di servernya Github dengan [Github Action](/post/auto-deploy-astro-di-github-pages.html) atau di Vercel maupun Netlify.
